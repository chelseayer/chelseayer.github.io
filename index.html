CS 184: Computer Graphics, Spring 2020

Final Project Proposal: Google Streetview to Panorama Painting

Wenlong Huang, Chelsea Ye, Mingmao Sun

Summary

This project attempts to solve the computational photography and imaging problem of “image-to-image translation” using CycleGAN. The result is demonstrated by converting Google street maps to panorama paintings that allows viewers to see the surrounding streetviews in oil painting style in 360 degrees.

Project Description

The core of this project is to convert images from one domain to another without pairing -- the problem is also known as unpaired image to image translation. The significant challenge is that it is hard to define direct correspondence between the images from the two domains, real photos and paintings, because the paired data are not abundant. To tackle this challenge, we use a famous work called CycleGAN, which enforces cycle-consistency in the translation between the two domains and leads to superior results in image generation.

Goals and Deliverables

  1. We PLAN to deliver: an interactive system in which users can drag the mouse to change the viewing angle and view the neighborhood in oil paintings. To accomplish this, we would have to create the oil paintings in panorama and create a system to enable the interactivity. However, due to the inconsistency of the performance of CycleGAN and the limitation of the quality of the data, the image quality may not be as good. Yet we plan to have at least a viewing system that has a baseline image quality that users would recognize that it is an oil painting of street view.
  2. We HOPE to deliver: If the data quality is great and everything goes well, users can view their neighborhood -- like Google Streetview -- that allows people to view their neighborhood as if it is an oil painting while seamlessly preserving the structure and the content of the image so that the buildings and the stores are easily recognizable.

Schedule

  • Week 1 - Data Collection: collect data from both Google Streetview and oil paintings by scraping them online
  • Week 2 - Model Training: train the CycleGAN model using online computing resources such as Google Cloud or AWS with the data collected
  • Week 3 - Creating Viewing System: create the interactive viewing system that allows users to drag their mouse and change the viewing angle.
  • Week 4 - Finalizing Reports and Buffer Time: finalize the report and some extra buffer time if anything does not go well.
